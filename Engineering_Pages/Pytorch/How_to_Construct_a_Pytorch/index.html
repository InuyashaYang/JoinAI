    深度学习框架:从微分图到反向传播¶

    为什么要学习框架原理?¶

    深度学习框架虽然使用便捷,但理解其底层机制对于深入掌握深度学习至关重要。通过学习框架原理,我们能: - 理解模型训练的本质过程 - 掌握自动微分的核心原理
    - 提升模型调试和优化能力

    微分图的本质¶

    微分图(Computational Graph)是深度学习框架的核心,它通过有向图结构来描述计算过程:

    基本组成¶

    1. 节点(Nodes):
    2. 代表具体的计算操作(如加法、乘法等)
    3. 存储计算结果和梯度信息
    4. 记录与其他节点的连接关系
    5. 边(Edges):
    6. 表示数据在节点间的流动
    7. 定义了计算的依赖顺序
    8. 指导梯度的反向传播

    工作机制¶

    微分图在训练过程中扮演双重角色: 1. 前向传播:按照边的方向执行计算 2. 反向传播:沿着边的反方向计算梯度

    接下来我们将使用 Python 和 NumPy 进行工程化实践。为了系统地实现一个基本的深度学习框架，从而支持微分图和反向传播算法，我们需要设计并实现多个类。以下是我们要实现的主要类列表及其简要描述：

    主要类列表¶

    1. Tensor 类
    2. 职责：
      • 存储数据（使用 NumPy 的 ndarray）和相应的梯度。
      • 记录创建该张量的操作，便于构建计算图。

    3. 主要属性：

      • data: 存储实际的数值数据。
      • grad: 存储相对于损失函数的梯度。
      • creator: 记录执行当前张量计算的操作节点。

    4. Operation 类（基类）

    5. 职责：
      • 定义所有操作节点的基础接口，包括前向计算和反向传播的方法。

    6. 主要方法：

      • forward(inputs): 执行前向计算，生成输出张量。
      • backward(gradient): 执行反向传播，计算输入张量的梯度。

    7. 具体操作类（继承自 Operation）

    8. 职责：
      • 实现具体的数学操作及其对应的梯度计算。

    9. 主要子类：

      • Add：
      • 功能：实现加法运算。
      • 前向：output = input_a + input_b
      • 反向：grad_a = grad_output * 1, grad_b = grad_output * 1
      • Multiply：
      • 功能：实现逐元素乘法运算。
      • 前向：output = input_a * input_b
      • 反向：grad_a = grad_output * input_b, grad_b = grad_output * input_a
      • MatMul：
      • 功能：实现矩阵乘法运算。
      • 前向：output = input_a @ input_b
      • 反向：grad_a = grad_output @ input_b.T, grad_b = input_a.T @ grad_output
      • ReLU：
      • 功能：实现 ReLU 激活函数。
      • 前向：output = max(0, input)
      • 反向：grad_input = grad_output * (input > 0)
      • 其他操作：
      • 根据需要，可以实现更多操作，如 Sigmoid、Softmax、Transpose 等。

    10. Graph 类

    11. 职责：
      • 管理和维护整个计算图的结构。
      • 组织节点和边，确保前向和反向传播的顺序。

    12. 主要方法：

      • add_operation(operation): 添加操作节点到计算图中。
      • forward(inputs): 执行前向传播，计算输出。
      • backward(loss_grad): 执行反向传播，计算梯度。

    13. Optimizer 类（基类）

    14. 职责：
      • 定义优化算法的基础接口，用于更新模型参数。

    15. 主要方法：

      • step(): 更新参数。
      • zero_grad(): 清零所有参数的梯度。

    16. 具体优化器类（继承自 Optimizer）

    17. 职责：
      • 实现具体的优化算法。

    18. 主要子类：

      • SGD（随机梯度下降）：
      • 功能：按学习率更新参数。
      • 实现：param -= learning_rate * param.grad
      • Adam（可选）：
      • 功能：实现 Adam 优化算法。
      • 实现：包括动量和自适应学习率的计算。

    19. Layer 类

    20. 职责：
      • 抽象神经网络中的层，管理参数和前向计算。
    21. 主要方法：
      • forward(inputs): 定义层的前向传播。
      • parameters(): 返回层的可训练参数。

    22. 主要子类：

      • Linear：
      • 功能：实现全连接层。
      • 参数：权重矩阵和偏置向量。
      • 前向：output = input @ weights + bias
      • 其他层：
      • 根据需要，可以实现更多层，如 Conv2D、Dropout 等。

    23. Model 类

    24. 职责：
      • 组合多个 Layer，构建整个神经网络模型。
      • 管理整体的前向和反向传播过程。

    25. 主要方法：

      • forward(inputs): 顺序调用各层的前向传播。
      • parameters(): 汇总所有层的可训练参数。

    26. Loss 类

    27. 职责：
      • 定义损失函数及其梯度计算。
    28. 主要方法：
      • forward(predictions, targets): 计算损失值。
      • backward(): 计算损失相对于预测值的梯度。
    29. 主要子类：
      • MeanSquaredError：
      • 功能：实现均方误差损失。
      • CrossEntropyLoss：
      • 功能：实现交叉熵损失。

    9. 实现优化器：

    12. 选择合适的损失函数，根据具体任务（如回归或分类）进行选择。
